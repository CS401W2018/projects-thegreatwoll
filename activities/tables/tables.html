When do you use a table?

We use a table on our webpage when we have a data that we want to present or show. Example of data could be numbers, schedules, names, etc. Anything that should be displayed in rows and columns.

When do you not use a table?

We do not use tables in our webpage's HTML for the purpose of layouts. Tables are not flexible and are not good to use when we are trying to create a responsive webpage. They can be used but it may disrupt the idea of creating a unified responsive webpage.

When do you need to do extra work for table accessibility?

We do extra work in making our tables accessible when we are trying to tie data within the tables to headers in the vertical axis or columns. We will have to create an ID for the cell in the column and connect it to the cell that contains the data.



Why don't we use the border attribute?

We do not use the border attribute in our HTML because it is a depricated function or way of modifying tables in our web page. It may also lead to errors when it comes to the validity of our HTML.

What is the easiest way to color every other row a different color?

The best way to accomplish this is by using the sudo function nth-child(even) on the table row tags in our CSS.
